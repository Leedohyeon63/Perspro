
===============================
던전 탐험 텍스트 RPG 게임입니다.  
플레이어는 여러 스테이지를 탐험하며 몬스터와 보스를 상대하고, 상점에서 아이템을 사고 장착하며 성장합니다.
게임 흐름이나 몬스터, 보스 이름은 할로우 나이트에서 따왔습니다.

주요 구성 요소

1. 게임 관리 (Game Flow)
- GameManager : 게임 전반을 관리합니다. (텍스트 RPG 실행, 스테이지 전환, 전투 이벤트, 보스 전투 관리)
- StageMap : 각 스테이지 맵의 구조와 특수 타일(벽, 상점, 보스, 포탈, 제단 등)을 관리
- Shop : 플레이어가 방문 가능한 상점 시스템. 아이템 구매 판매 및 장착 지원.

2. 전투 시스템
- Battle (추상 클래스) : 공격 및 피해 처리의 기본 틀 제공.
- Actor : 플레이어 및 몬스터의 공통 기반 클래스. 체력, 공격력, 상태 이상 관리.
- Monster : Actor를 상속받은 몬스터 전용 클래스. 공통 인터페이스 제공.
- BattleManager : 일반 몬스터 전투 및 보스 전투를 제어. 가드, 스킬 사용, 아이템 사용 처리.

3. 플레이어
- Player : 플레이어 캐릭터의 체력, 공격력, 오브(회복 자원), 골드, 위치를 관리.
- Inventory : 인벤토리 관리 클래스. 아이템 장착 및 해제, 소모품 사용, 보스 보상 지급, 전투 아이템 사용.

4. 몬스터와 보스
- Mosterlist.h에 모든 몬스터와 보스 클래스 정의 포함.
- 일반 몬스터 
 스테이지당 2~3마리 별도의 몹이 존재
 일부 몬스터는 독자적인 스킬 사용
- 보스 몬스터 :
  나이레스, 토르비오, 위대한 그롤, 위대한 어머니 실크, 풀려난 자
 각 보스는 독자적인 패턴과 아이템 연계 효과를 보유.

게임 흐름
1. 시작 : 플레이어는 GameManager::TextRpg()을 통해 게임을 시작합니다.
2. 탐험 : StageMap을 이동하며 벽, 상점, 포탈, 보스, 제단 등을 탐험합니다.
3. 전투 : 랜덤하게 몬스터 전투가 발생합니다. 보스 타일에서는 보스 전투가 진행됩니다.
4. 아이템 관리 : 인벤토리와 상점을 통해 아이템을 구매, 사용, 장착합니다.
5. 스테이지 진행 : 포탈을 통해 다음 스테이지로 이동합니다. 보스를 처치해야만 진행할 수 있습니다.
6. 클리어 조건 : 최종 보스를 처치하고 보상 아이템을 획득하면 게임이 클리어됩니다.
